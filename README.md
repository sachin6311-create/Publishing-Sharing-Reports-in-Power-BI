# Publishing-Sharing-Reports-in-Power-BI
Publishing &amp; Sharing Reports in Power BI

Q1. Explain why publishing and sharing reports is considered more important
than just building dashboards in Power BI Desktop.
Publishing and sharing reports is more important because it enables business users to
access insights and make decisions.
While dashboards created in PowerBI Desktopare useful, they only provide value when
shared.
Sharing improves collaboration, ensures real-time access, and allows stakeholders to
interact with reports.
Without publishing, reports remain limited to the creator and do not contribute to business
outcomes.

Q2. Define Power BI Service and explain its role in comparison to Power BI
Desktop in the reporting lifecycle.
PowerBIDesktopisatoolused forcreating reports, transforming data, and building data
models.
PowerBIService is a cloud-based platform used for publishing, sharing, and managing
reports.
In the reporting lifecycle, Desktop is used for building reports, whereas Service is used for
distributing and collaborating on them.

Q3. What does publishing a report mean in Power BI? List any two outcomes
that occur after a report is published.
Publishing a report means uploading it from PowerBI Desktopto Power BIService.
After publishing:
1. Thereport becomes accessible online through the Power BI Service.
2. It can be shared with other users for viewing and collaboration.

   
Q4. List and explain the steps involved in publishing a report from Power BI
Desktop to Power BI Service in the correct sequence.
1. Create the report in Power BI Desktop.
2. Save the .pbix file.
3. Click on the Publish button.
4. Sign in to your Power BI account.
5. Select the desired workspace.

   
Q5. Differentiate between Report and Dashboard in Power BI based on
structure and usage.
AReportis a multi-page document withdetailed data visualizations and high interactivity.
ADashboardisasingle-page summary thatcombines visuals from multiple reports.
Reports are mainly used for in-depth analysis, while dashboards are used for quick insights
and monitoring.


Q6. Explain the difference between Direct Report Sharing, Workspace Access,
and Power BI Apps. Also mention one use case for each.
Direct Report Sharing allows users to share reports directly with specific individuals. It is
useful for small teams.
WorkspaceAccess provides role-based access to a group of users for collaboration. It is
useful for project teams.
PowerBIAppsallowreportsto be packaged and distributed toa large audience. It is useful
for organization-wide reporting.


Q7. Describe the four workspace roles available in Power BI and explain how
access control is managed using these roles.
The four roles are Admin, Member,Contributor, and Viewer.
Adminhasfull control over the workspace.
Membercaneditandpublish content.
Contributor can create and modify content but has limited permissions.
Viewer canonly viewreports.
Access control is managed by assigning appropriate roles to users, ensuring data security
and proper governance.

Q8. Why arePower BIApps considered more secure and scalable than direct
sharing? Explain from a governance and enterprise usage perspective.
PowerBIAppsaremoresecurebecausetheyprovidecontrolled access and prevent
unauthorized modifications.
They arescalable as they allow distribution to a large number of users efficiently.
Fromagovernanceperspective, Apps ensure consistent reporting, centralized management,
and better version control in enterprises.

Q9. Explain Row-Level Security (RLS) in Power BI. How does RLS ensure that
different users see different data using the same report?
Row-Level Security (RLS) is a feature that restricts data visibility based on user roles.
It works by defining filters on datasets and assigning users to specific roles.
Whenusersaccessthe report, they only see the data allowed by their assigned role,
ensuring data privacy and security.

Q10. Discuss any three risks if reports are shared without Apps, workspace
roles, or RLS, and explain how Power BI best practices prevent them.
1. Data Security Risk: Sensitive data may be exposed to unauthorized users.
2. Lack of Governance: Nocontrol over access and editing permissions.
3. Data Misuse: Users may misinterpret or misuse data.
These risks can be prevented by implementing Power BI Apps,workspaceroles, and Row
Level Security to ensure proper access control and governance.
